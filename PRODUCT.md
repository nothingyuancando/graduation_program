# Product

## Register

product

## Users

The primary users are students preparing coursework, exams, graduation projects, or technical interviews. They usually already have learning materials, but need help turning scattered content into a trackable learning process. Their daily context is task-focused: set a goal, import or write material, verify understanding, review weak points, and see whether mastery is improving.

## Product Purpose

This is an AI personalized learning notes platform built around a learning loop. It is different from tool-first note apps such as Notion or Obsidian: it should not only store knowledge, but actively drive the user from input to mastery. Success means a user can define what they want to learn, convert materials into structured notes, explain concepts in their own words, practice recall, expose weaknesses, and update the next learning plan from mastery data.

## Brand Personality

Clear, reliable, and motivating. The interface should feel like a calm learning cockpit: structured enough for serious study, warm enough to reduce anxiety, and decisive enough to guide the next action.

## Anti-references

Avoid generic AI SaaS landing-page patterns, decorative gradients, vague "AI assistant" copy, excessive card grids, and upload-centered flows that make the product look like a batch document parser. Avoid playful visuals that weaken trust, and avoid dense knowledge-management UI that makes the learner decide everything alone.

## Design Principles

1. Make the learning loop visible in every core workflow.
2. Treat AI output as a step toward verification, not the final destination.
3. Prefer next actions over feature catalogs.
4. Keep upload auxiliary and explain processing state honestly.
5. Surface weak points and mastery signals as practical guidance.

## Accessibility & Inclusion

Use readable Chinese UI copy, high contrast, visible focus states, and stable layouts across mobile and desktop. Do not rely on color alone for learning state. Keep motion subtle and state-driven, and respect reduced-motion expectations where animations are not essential.
